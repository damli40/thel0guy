your docs have a dollar cost per integration. i put a number on it.

a growing share of your integrations start with a developer pointing claude code or cursor at your docs. when the docs are ambiguous, the agent doesn't bounce. it grinds, burns tokens, and sometimes ships the wrong config. nobody on your team can see this happening. the audit makes it visible, in dollars and minutes, measured against your own test suite.

what you get

i run your toolchain through a controlled multi-arm agent harness: the same model builds the same real integration task with and without your docs in context, multiple runs per arm. pass or fail is decided by rerunning your test suite in a clean container. what the model claims is not evidence.

  • a measured table: cost, wall clock, turns, and tool calls per working integration, per arm
  • the gap map: which questions your docs answer, which ones send the agent digging through source
  • a prioritized fix list, specific enough to hand to whoever writes your docs
  • a one-hour walkthrough call, findings first, no slides

every number in the report comes from the provider's own billing fields, summed. measured, not estimated.

the method is public

i built the harness by running a 40-run controlled experiment on a smart-contract toolchain. four arms, same prompt, test-suite oracle. the short version:

setupmedian cost / working integrationmedian time
bare model$0.57311s
+ retrieval tool$0.10156s
+ curated docs$0.0960s
+ docs and retrieval$0.0970s

bad context cost about 6x per working integration and 5x the clock. on a second task chosen outside the docs' coverage, docs alone lost nearly all their edge while docs plus retrieval stayed cheapest and fastest. that boundary is exactly what the audit finds in your docs. a full public sample audit of a well-known toolchain is coming; ask and i'll send the current writeup.
